## Análise Técnica

### Estado Atual
Projeto Laravel 12 recém-criado sem implementação de POS. Estrutura padrão sem:
- Models de comanda/pedido
- Sistema de impressão
- Frontend de PDV

### Problema
Conectar **VPS (IP Público)** com Laravel PDV às **impressoras locais (IP Privado)** conectadas via USB ao PC local.

### Restrições
1. VPS não pode iniciar conexão direta com PC local (NAT/Firewall)
2. Impressoras térmicas usam protocolo ESC/POS
3. Impressão deve ser quase instantânea
4. Não pode perder comandas se conexão cair

### Solução Arquitetural
**Print Server Local com Polling via API REST**

**Por quê?**
- ✅ Funciona atrás de NAT/firewall (agente inicia conexão)
- ✅ Simples de implementar e debugar
- ✅ Confiável (retry automático, jobs persistidos)
- ✅ Baixa complexidade para restaurante

**Alternativas descartadas:**
- WebSocket: Mais complexo, requer manutenção de conexão persistente
- VPN/Tunnel: Complexidade de infraestrutura desnecessária

## Plano de Implementação

### Arquitetura

```
[PDV Web na VPS] 
    ↓ Cria Order
CreateOrderAction → CreatePrintJobAction
    ↓
PrintJob (status: pending) salvo no DB
    ↓
[Agente Local no PC] → Polling GET /api/print-jobs/pending (a cada 3s)
    ↓
Recebe PrintJob → Imprime via ESC/POS
    ↓
POST /api/print-jobs/{id}/complete
    ↓
PrintJob (status: completed)
```

### Riscos Técnicos e Mitigações

**1. Compatibilidade de Impressoras**
- **Risco**: Diferentes modelos ESC/POS podem ter comandos variados
- **Mitigação**: Usar `node-thermal-printer` (suporta Epson, Star, Daruma, etc.)

**2. Latência de Polling**
- **Risco**: Delay entre criar comanda e imprimir
- **Mitigação**: Polling configurável (padrão: 3 segundos), aceitável para restaurante

**3. Falha de Rede Temporária**
- **Risco**: Perder jobs se agente ficar offline
- **Mitigação**: Jobs ficam em fila (status: pending) até serem completados, retry infinito

**4. Segurança do Token**
- **Risco**: Token de acesso do agente vazado
- **Mitigação**: Laravel Sanctum, HTTPS obrigatório, token por impressora (revogável)

**5. Múltiplas Impressoras**
- **Risco**: Cozinha, bar, caixa precisam de impressoras diferentes
- **Mitigação**: Campo `type` em Printer, agente filtra jobs por `printer_id`
