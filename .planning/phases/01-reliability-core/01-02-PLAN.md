# Phase 1 Plan 2: Circuit Breaker for API Polling

## Objective

Implementar circuit breaker pattern no polling loop do print agent para detectar quando a API Laravel está offline e pausar requisições temporariamente. Isso evita sobrecarga da API com requests falhados e reduz logs desnecessários durante downtime.

## Context

**Requirements addressed:** REQ-021  
**Phase goal contribution:** Previne sobrecarga da API durante downtime e melhora resiliência do sistema.

**Current behavior:** Agent faz polling a cada 3s mesmo quando API está offline, gerando logs de erro contínuos.

**Desired behavior:** Após N falhas consecutivas, circuit breaker "abre" e pausa polling por período crescente. Tenta reconectar periodicamente até API voltar.

## Tasks

### Task 1: Create circuit breaker module

**Files:** `print-agent-desktop/agent/circuit-breaker.js`

**Action:**
Criar módulo circuit breaker com 3 estados:
- **CLOSED:** Normal operation, requests passam
- **OPEN:** API offline detectada, requests bloqueados
- **HALF_OPEN:** Testando reconexão, permite 1 request

**Implementação:**
```javascript
class CircuitBreaker {
  constructor(failureThreshold = 5, resetTimeout = 30000) {
    this.state = 'CLOSED';
    this.failureCount = 0;
    this.failureThreshold = failureThreshold;
    this.resetTimeout = resetTimeout;
    this.nextAttempt = null;
  }

  async execute(fn) {
    if (this.state === 'OPEN') {
      if (Date.now() < this.nextAttempt) {
        throw new Error('Circuit breaker is OPEN');
      }
      this.state = 'HALF_OPEN';
    }

    try {
      const result = await fn();
      this.onSuccess();
      return result;
    } catch (error) {
      this.onFailure();
      throw error;
    }
  }

  onSuccess() {
    this.failureCount = 0;
    this.state = 'CLOSED';
  }

  onFailure() {
    this.failureCount++;
    if (this.failureCount >= this.failureThreshold) {
      this.state = 'OPEN';
      this.nextAttempt = Date.now() + this.resetTimeout;
    }
  }

  getState() {
    return {
      state: this.state,
      failureCount: this.failureCount,
      nextAttempt: this.nextAttempt
    };
  }
}
```

**Verify:**
```bash
node -e "const {CircuitBreaker} = require('./print-agent-desktop/agent/circuit-breaker.js'); const cb = new CircuitBreaker(3, 5000); console.log(cb.getState());"
```

**Done when:**
- Arquivo `circuit-breaker.js` existe e exporta `CircuitBreaker`
- Classe implementa 3 estados (CLOSED, OPEN, HALF_OPEN)
- Transições de estado funcionam corretamente
- Teste manual mostra estado inicial CLOSED

### Task 2: Integrate circuit breaker into polling loop

**Files:** `print-agent-desktop/agent/index.js`

**Action:**
Integrar circuit breaker no polling loop principal:

1. Importar `CircuitBreaker` no topo do arquivo
2. Criar instância: `const apiCircuitBreaker = new CircuitBreaker(5, 30000)`
   - 5 falhas consecutivas → OPEN
   - 30s timeout antes de tentar reconectar
3. Wrap chamada `apiClient.getPendingJobs()` com circuit breaker:
   ```javascript
   try {
     const jobs = await apiCircuitBreaker.execute(() => 
       apiClient.getPendingJobs(config.PRINTER_IDENTIFIER)
     );
     // processar jobs...
   } catch (error) {
     if (error.message === 'Circuit breaker is OPEN') {
       logger.warn('Circuit breaker OPEN - API offline, pausing polling');
       // não logar erro completo, apenas status
     } else {
       logger.error('Error fetching jobs:', error.message);
     }
   }
   ```
4. Logar mudanças de estado do circuit breaker:
   - CLOSED → OPEN: "API offline detected, pausing polling for 30s"
   - OPEN → HALF_OPEN: "Testing API reconnection..."
   - HALF_OPEN → CLOSED: "API back online, resuming normal polling"
   - HALF_OPEN → OPEN: "API still offline, extending pause"
