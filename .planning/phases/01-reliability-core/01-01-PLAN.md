# Phase 1 Plan 1: Retry Logic for Print Failures

## Objective

Implementar retry logic com backoff exponencial para falhas de impressão no print agent. Jobs que falham por problemas temporários (rede, impressora offline, arquivo travado) serão automaticamente retentados até 3 vezes antes de serem marcados como falhos permanentemente.

## Context

**Requirements addressed:** REQ-013  
**Phase goal contribution:** Garante que jobs não sejam perdidos por falhas temporárias, aumentando confiabilidade do sistema em produção.

**Current behavior:** Quando uma impressão falha, o job é perdido e precisa ser recriado manualmente.

**Desired behavior:** Falhas temporárias são retentadas automaticamente com delays crescentes (1s, 2s, 4s).

## Tasks

### Task 1: Create retry utility module

**Files:** `print-agent-desktop/agent/retry-util.js`

**Action:**
Criar módulo utilitário para retry logic reutilizável:
- Função `retryWithBackoff(fn, maxRetries = 3, baseDelay = 1000)`
- Backoff exponencial: delay = baseDelay * (2 ^ attempt)
- Logging de cada tentativa com attempt number
- Retornar resultado ou throw após max retries
- Capturar e logar stack trace apenas no último erro

**Verify:**
```bash
# Criar teste manual
node -e "const {retryWithBackoff} = require('./print-agent-desktop/agent/retry-util.js'); retryWithBackoff(() => { throw new Error('test'); }, 3, 100).catch(() => console.log('OK'))"
```

**Done when:**
- Arquivo `retry-util.js` existe e exporta `retryWithBackoff`
- Função implementa backoff exponencial corretamente
- Logs mostram attempt number e delay
- Teste manual funciona

### Task 2: Wrap print operations with retry logic

**Files:** 
- `print-agent-desktop/agent/printer.js`
- `print-agent-desktop/agent/pdf-printer.js`
- `print-agent-desktop/agent/html-to-pdf.js`
- `print-agent-desktop/agent/windows-printer.js`

**Action:**
Aplicar retry logic nas funções de impressão:

**printer.js:**
- Importar `retryWithBackoff`
- Wrap `printJob()` com retry (3 tentativas, 1s base delay)
- Logar job ID e tipo de conteúdo em cada retry
- Se todas tentativas falharem, logar erro final e retornar null

**pdf-printer.js:**
- Wrap `printPDF()` com retry (3 tentativas, 1s base delay)
- Retry cobre falhas de SumatraPDF (arquivo travado, impressora offline)

**html-to-pdf.js:**
- Wrap `convertHTMLtoPDF()` com retry (2 tentativas, 500ms base delay)
- Retry cobre falhas de Puppeteer (timeout, crash)

**windows-printer.js:**
- Wrap operações de impressão térmica com retry (3 tentativas, 1s base delay)
- Retry cobre falhas de conexão TCP/USB

**Verify:**
```bash
cd print-agent-desktop
npm start
# Simular falha desconectando impressora
# Verificar nos logs que retry acontece 3x antes de falhar
```

**Done when:**
- Todas funções de impressão usam `retryWithBackoff`
- Logs mostram retry attempts com delays crescentes
- Jobs não são perdidos por falhas temporárias
- Teste manual com impressora offline mostra 3 retries

### Task 3: Update job completion logic

**Files:** `print-agent-desktop/agent/index.js`

**Action:**
Atualizar lógica de marcação de jobs como completed/failed:

- Se `printJob()` retorna sucesso → marcar como `completed` (comportamento atual)
- Se `printJob()` retorna null após retries → logar erro mas NÃO marcar como completed
- Adicionar contador de falhas por job (max 3 ciclos de polling)
- Após 3 ciclos falhados, logar "Job permanently failed" e pular
- Adicionar métrica: "Jobs retried successfully" vs "Jobs permanently failed"

**Verify:**
```bash
cd print-agent-desktop
npm start
# Criar job de teste
# Desconectar impressora
# Verificar que job não é marcado como completed
# Reconectar impressora
# Verificar que job é impresso no próximo polling
```

**Done when:**
- Jobs com falha temporária não são marcados como completed
- Jobs são retentados em próximos ciclos de polling
- Após 3 ciclos falhados, job é pulado permanentemente
- Logs mostram claramente status de retry
